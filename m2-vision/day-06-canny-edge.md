## 0. 预备新概念
- 边缘 = 图像里亮度突然变化的地方
- 梯度 = 变化的快慢和方向 

## 1. 什么是边缘【必懂】
- 本质: 相邻像素亮度差很大的位置
- 例子: 30,30,30,200,200 → 突变处是边缘
- 用途: 智能车循迹 / 车道检测 / 物体轮廓

## 2. 什么是梯度【必背】
- 直觉: 像素亮度变化的快慢
- 算法: 相邻像素相减 (差大 = 梯度大 = 边缘)
- 方向: 水平梯度 Gx (检测竖直边) + 垂直梯度 Gy (检测水平边)
- 边缘强度 = sqrt(Gx^2 + Gy^2)
- 地位: 边缘检测 / 特征点 / CNN / 光流 都用梯度

## 3. Canny 四步【必懂】
1. 高斯滤波降噪 (Day 5)
2. 计算梯度
3. 非极大值抑制 (细化边缘)
4. 双阈值筛选 + 连接

## 4. 双阈值参数【必背】
- 用法: cv2.Canny(img, 低阈值, 高阈值)
- 三类: 大于高 = 强边缘, 中间 = 弱边缘(连强边才留), 小于低 = 丢弃
- 经验: 高约等于低的 2-3 倍, 起点 (50, 150)
- 调参:
  - 边缘太杂 → 提高阈值
  - 边缘断裂 → 降低阈值 (优先降低阈值那个)

## 5. 核心代码模式【必懂】
`edges = cv2.Canny(img,50,150)`

标准流程: 灰度 → 高斯 → Canny
```python
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
blurred = cv2.GaussianBlur(gray, (5, 5), 0)
edges = cv2.Canny(blurred, 50, 150)
```

## 6. 思考题闭环
### Q1: 梯度与边缘关系
我的答案: 梯度大说明像素变化快, 像素差大的地方就是边缘, 因为边缘是亮度突变的地方

### Q2: 边缘断裂调参
我的答案: 调低阈值, 边缘丢失说明本该是边缘的没到达阈值被丢了

### Q3: Canny 输入类型 bug
我的答案: 要先转灰度, 最好再高斯滤波处理一下

## 7. 必背【追加 2 条】
1. Canny 双阈值: 高是低的 2-3 倍, (50,150) 起步
2. 标准流程: 灰度 → 高斯 → Canny

## 8. 易错点
| 坑 | 后果 |
|---|---|
| Canny 直接喂彩色图 | 应先转灰度 |
| 阈值设太高 | 边缘丢失 |
| 不先降噪 | 假边缘多 |
