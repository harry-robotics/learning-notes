## 学习方法演化记录

### v1.0 — 启动期 (Day 1-2)
- 一开始想"什么都学下来", 笔记里堆满细节
- 教训: 信息过载, 没有重点

### v1.1 — 引入"四象限分类" (Day 3 起)
每天结束按四个象限分类:
- 操作必须熟练
- 操作跟一遍即可
- 知识重点记忆
- 知识不必死记

效果: 认知带宽分配清晰多了

### v2.0 — 关键转折: AI 时代学习论的觉醒 (Day 7)
**这是一次重要的方法论修正**。

#### 我当时的困惑
- 学了 Python 一天就过完基础语法
- 同时学 C 语言的指针感觉接受不来
- 开始怀疑: AI 都能生成代码, 我学这些有意义吗?

#### 我得出的结论
1. **AI 时代写代码贬值, 懂代码升值**
2. **要背的是"判断标准"和"工程纪律", 不是 代码本身**
3. **AI 是脚手架, 不是替代品** — 我要建立"内部模型", 这样才能判断 AI 写的对不对
4. **debug 能力 > 默写能力**

#### 调整后的"必背"清单 (全 M1 阶段控制在 10 条)
不再是默写代码, 而是判断标准:
1. 中断三铁律
2. millis 减法形式 (避免溢出 bug)
3. WiFi.begin() 非阻塞 + 超时降级思维
4. begin 必有 end (资源配对纪律)
5. 指针 5 条铁律
6. HTTP 状态码反推 debug 层级
7. struct tm 两大坑
8. PWM 三参数权衡
9. ESP32 双核分工
10. JSON 嵌套访问思路

#### 行动调整
- 不再默写代码
- 看到 AI 写的代码能 debug 即可
- 重心: 判断力 > 记忆力

## 道心动摇时刻 / 思考记录

### 时刻 1: "AI 都能写代码我学这些有意义吗?" (Day 7)
- 触发: Python 学得快, C 指针学得累, 同时怀疑学习的意义
- 解决: 想清楚 AI 时代的能力分层 — 写代码贬值, 判断力升值
- 结论: 继续学, 但调整学习方法 (见 v2.0)

### 时刻 2: "ESP32 对我未来方向有用吗?" (Day 6)
- 担忧: 学嵌入式底层会不会浪费时间, 跟自动驾驶/具身智能脱节?
- 思考: 列出每个知识点的未来用途
- 结论: 直接用上的 (millis 思维 / 中断 / 指针 / 双核思维) 比想象多;
        即使具体 API 用不上, 嵌入式思维方式一辈子受用

## 学习心得

  - 2026-5-24：
  - 我这一周一边学习Esp32的C++，一边还学习CMC的Python，一时间感觉很难，压力好大。于是我就自我反思，该怎么提高学习效率，我就发现，其实现在再Esp32里学到的许多复杂代码，未来换了一个新的嵌入式开发平台之后，就用不上了，而且AI时代就应该主动让AI发挥价值。所以我决定，调整未来的学习模式，我真正要掌握的，不是代码本身，而是工程师理解代码，debug的能力，工程师核心思维以及实物调试能力。在此基础上我将优化我的学习节奏和学习路径。
