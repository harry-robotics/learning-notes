## 1. ESP32 是什么?

ESP32是乐鑫科技公司推出的：集成了WiFi,蓝牙，双核处理器的低功耗物联网芯片。

## 2. ESP32 vs Arduino UNO 三大关键差异

### 2.1 电压差异

Arduino UNO默认使用5v电压；ESP32 是3.3v，给5v会烧坏
用到逻辑电平转换模块把Arduino的5v传感器接到ESP32上

### 2.2 无线通信

ESP32相比于Arduino内置了蓝牙，WiFi,使得项目可以远程操控，会运用在后续升级我的智能小车

### 2.3 双核处理

ESP32相比于Arduino，内置双核处理系统，即可以同时处理两件事，而不像Arduino只能轮流做

## 4. 概念校准: Arduino 时期的错误理解

| 概念 | 我之前的错误理解 | 正确理解 |
|---|---|---|
| digitalWrite() | 输入  | 设置引脚电平，即告诉引脚变高/低 |
| PWM | 模拟信号 | 本质是数字信号，通过改变高电平持续的比例（占空比）来模拟出模拟信号的效果 |

> **Note**:比如LED对PWM的反应会看起来亮度在变化，其实是因为人眼无法分辨快速闪烁；电机对PWM的反应是转速变化，是因为电机有惯性，跟不上快速的通断

## 5. I2C，单总线和普通GPIO

I2C全称集成电路间通信，是一种两根线就让多个设备之间通信的协议（只有0和1）
两根线：SDA = Serial Data(数据线)
       SCL = Serial Clock(时钟线),实现多个设备共用

单总线：通过一根线传输复杂数据（用到时序协议）
例子：DHT22 温湿度传感器

普通GPIO: 只传一个状态（开和关）即HIGH(1)和LOW(2)
例子：控制继电器开关，读取按键状态

## 6. 温控风扇项目的"系统框架"

### 信号流图

```text
        ┌──────────────────────────────────────┐
        │            ESP32 (核心)              │
        │                                      │
        │   1. 读温度 → 2. 判断 → 3. 控制       │
        └──┬──────────────┬──────────────┬─────┘
           │              │              │
   ┌───────▼─────┐  ┌─────▼─────┐  ┌────▼──────┐
   │   DHT22     │  │   OLED    │  │  继电器    │
   │  (温湿度    │  │  (显示    │  │  +风扇     │
   │   传感器)   │  │   信息)   │  │           │
   └─────────────┘  └───────────┘  └───────────┘
        │                │                │
    单总线协议      I2C协议          普通GPIO
    (1根数据线)    (2根线: SDA+SCL)  (1根控制线)
        │                │                │
   GPIO 4(示例)    GPIO 21+22       GPIO 13
```

### 系统工作流程

循环执行:
1. ESP32 通过单总线协议,从 DHT22 读取温度数据
   (数据是数字化的,直接得到一个数字,比如 26.5°C)
   
2. ESP32 判断温度是否超过阈值(比如 28°C)
   - 是 → 通过 GPIO 输出 HIGH,继电器导通,风扇通电
   - 否 → 通过 GPIO 输出 LOW,继电器断开,风扇断电
   
3. ESP32 通过 I2C 协议,把当前温度和风扇状态显示到 OLED
   (类似"温度: 26.5°C  风扇: ON")
   
4. 等待1秒,回到第1步
