## 1. Serial 串口通信深化

### 1.1 ESP32 vs Arduino 的波特率差异
- Arduino 是9600波特率
- Esp32 是115200（可以更高）
>> 波特率：串口通信的传播速度

### 1.2 串口缓冲区(RX/TX)的原理
- RX:接收，数据存储到缓冲区(Serial.read)
- TX:发送，数据存储到缓冲区(Serial.print)

### 1.3 关键 API
| 函数 | 作用 | 何时用 |
|---|---|---|
| Serial.available() | 返回 RX 缓冲区还有多少字节 | 读之前先检查 |
| Serial.read() | 读一个字节 | 配合 available 用 |
| Serial.readString() | 读到换行符为止 | 接收文本命令时 |
| Serial.flush() | 等 TX 缓冲区清空 | 关键时刻确保数据发完 |
| Serial.setRxBufferSize(1024) | 改 RX 缓冲区大小 | 高速接收时 |


### 1.4 缓冲区导致的两类常见 bug
- 1.缓冲区溢出场景
RX,TX缓冲区都是默认256字节，超过则新数据直接丢失
- 修复方法：调大缓冲区：Serial.setRxBufferSize(1024)；
用更高优先级的任务读取；用中断及时清空
- 2.Serial.println卡顿
当TX缓冲区充满时，CPU会等待TX缓冲区清空，等待期间CPU什么都做不了（代价所在）
- 优化方法：升高波特率，使得打印速度更快，大幅降低阻塞时间；
一次打印多个值；限制打印频率，而不是连着打印；用FreeRTOS任务专门处理串口（以后学）


## 2. ESP32 的三个 UART
>>**UART**:专门负责串口通信的硬件模块

### 2.1 Serial / Serial1 / Serial2 的区别
Serial   → 默认连 USB-CH340,写代码看到的"串口监视器"就是它
Serial1  → 独立 UART,默认引脚有限制,需要手动指定 GPIO
Serial2  → 独立 UART,默认 GPIO 16(RX) + GPIO 17(TX)


### 2.2 为什么需要多个 UART(实际场景)
场景:你的智能小车要同时:

- 1.通过 USB 连电脑调试(Serial)
- 2.通过蓝牙模块和手机通信(Serial1)
- 3.通过 GPS 模块获取位置(Serial2)


### 2.3 代码模板

```c
void setup() {
  Serial.begin(115200);                          // 默认 UART,USB 调试
  Serial1.begin(9600, SERIAL_8N1, 16, 17);       // UART1,自定义引脚
  Serial2.begin(38400, SERIAL_8N1, 26, 27);      // UART2,自定义引脚
}

void loop() {
  if (Serial1.available()) {           // 蓝牙数据来了
    char c = Serial1.read();
    Serial.println(c);                 // 转发到 USB 调试
  }
}
```


## 3. ESP32 双核架构

### 3.1 Core 0 和 Core 1 的分工
| 核 | 功能 |
|---|---|
| Core 0 | 跑系统底层（WiFi、蓝牙的协议栈）|
| Core 1 | 跑你写的应用代码 |



### 3.2 默认情况你的代码在哪个核心
代码默认跑在 Core 1(APP_CPU) 上。
但Core 0 默默跑着 WiFi 和蓝牙的底层协议——不需要管它,但它一直在工作


### 3.3 双核相比单核的实际意义
双核并行vs任务切换
两个CPU同时执行不同的代码vs一个CPU轮流执行多个任务，快速切换(ms级)
- >>ESP32具备的两种能力：
- 1. 双核真正同时跑不同任务
- 2. 单核内部又能用FreeRTOS任务切换跑多个任务


## 4. FreeRTOS 任务概念
FreeRTOS实时操作系统，内置在ESP32

### 4.1 单任务 vs 多任务的本质差异
- 单任务：按照顺序执行任务
- 多任务：FreeRTOS帮忙管理“何时切换，切换到哪个任务”(任务调度)


### 4.2 任务的三要素(优先级 / 栈 / 状态)
| 优先级 | 栈 | 状态 |
|---|---|---|
| 数字越大越优先 | 任务自己的内存 | 运行中 / 阻塞 / 挂起 |

## 5. 回答Day2疑难总结
- 1. 中断和轮询场景的应用判断：中断是有外部突发事件 + 必须立即响应（比如急停）；轮询是我在主动控制/周期性查询（会有ms级的慢点）
- 2. 呼吸灯中PWM分辨率改变改成16的影响：PWM三个核心参数：通道，频率，分辨率
- 频率：对应人眼看到灯光的效果， LED在5k赫兹对应最佳区间
- 分辨率（单位bit）：占空比可以分为多少级，人眼对亮度变化的便是极限在200到300级左右
>>频率和分辨率：两者之间有一个公式转化
最大频率=80M/2^分辨率（这决定了两者之间的矛盾关系）
